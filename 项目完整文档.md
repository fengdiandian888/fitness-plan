## 1. 项目概述

### 1.1 核心功能

| 功能 | 描述 |
|------|------|
| 每日计划 | 天气感知 + 时间轴 + 训练动作清单 + 晚餐配比 |
| 体重追踪 | Canvas 画折线图 + 里程碑进度条 |
| 训练跟练 | 三分化训练 + B站视频嵌入 + 智能计时器 |
| 视频管理 | 本地视频文件管理，支持拖拽上传 |
| 云端同步 | GitHub Gist 存储，跨设备数据共享 |
| PWA 支持 | 可添加到手机主屏幕，离线可用 |

### 1.2 技术栈

```
前端：HTML5 + CSS3 + Vanilla JavaScript（零框架依赖）
数据：localStorage（本地）+ GitHub Gist API（云端）
媒体：B站嵌入视频 + 本地视频文件
天气：wttr.in 免费 API
部署：GitHub Pages（静态托管）
```

| 规范 | 说明 |

| 手机优先 | max-width: 500px |

### 3.2 视频管理后台.html - 视频管理系统

**路径：** `视频管理后台.html`

**功能特性：**

| 功能 | 说明 |
|------|------|
| 动作列表 | 自动收集所有训练动作 |
| 拖拽上传 | 拖拽视频文件到对应动作 |
| 本地存储 | 视频数据存储在 localStorage |
| 视频预览 | 支持预览已添加的视频 |
| 统计面板 | 显示总动作数、已添加视频数 |

**数据结构：**

```javascript
// localStorage 键名
const STORAGE_KEY = 'fitness_videos';

// 数据结构
{
  '动作名称': {
    blob: 'data:video/...',  // Base64 编码的视频数据
    name: 'video.mp4',      // 原始文件名
    size: 1024000,          // 文件大小(bytes)
    type: 'video/mp4',      // MIME 类型
    time: 1750000000000     // 上传时间戳
  }
}
```

**使用流程：**

1. 打开视频管理后台
2. 选择训练计划 Tab（如：拉日训练）
3. 找到对应动作
4. 拖拽视频文件到动作的虚线框
5. 点击保存
6. 可点击"预览"查看视频效果

**限制：**

- 单个视频最大 50MB
- 支持格式：mp4、webm、mov
- 数据存储在浏览器本地，清除缓存会丢失

**路径：** `减脂训练日跟练_三分化.html`

**核心功能：**

| 功能 | 说明 |
|------|------|
| 智能计时器 | 自动训练计时 + 休息计时 |
| 动作切换 | 点击底部动作按钮跳转到对应动作 |
| 视频播放 | 加载本地视频并静音循环播放 |
| 语音播报 | 使用 Web Speech API 播报动作名称 |
| 进度追踪 | 显示组内进度 + 整体训练进度 |

**计时器状态机：**

```
idle → training → rest → training → ... → finished
         ↓
      paused → training
```

**关键函数：**

```javascript
// 全局计时器对象
window.TrainingTimer = {
  // 初始化
  init(moves, startIndex, title),
  
  // 开始训练
  start(),
  
  // 完成本组
  completeSet(),
  
  // 跳过休息
  skipRest(),
  
  // 切换动作（点击底部按钮）
  jumpToMove(index),
  
  // 暂停/继续
  pause(),
  resume(),
  
  // 退出
  exit()
};
```

## 4. 视频管理系统

### 4.1 系统架构

```
┌─────────────────────────────────────────┐
│           视频管理后台.html               │
│  ├── 读取 data.js 中的 FIT.plans        │
│  ├── 收集所有训练动作名称                │
│  ├── 渲染动作列表 UI                     │
│  └── 绑定拖拽事件                        │
├─────────────────────────────────────────┤
│         localStorage                     │
│  └── fitness_videos: {                  │
│        '动作名称': { blob, name, ... }  │
│      }                                   │
├─────────────────────────────────────────┤
│         训练页面                         │
│  └── TrainingTimer.loadMove()           │
│      └── TrainingTimer.getLocalVideo()   │
│          └── 返回本地存储的视频对象        │
└─────────────────────────────────────────┘
```

### 4.2 视频流程

1. **上传阶段**
   - 用户拖拽视频文件
   - 转换为 Base64 Blob
   - 存储到 localStorage

2. **加载阶段**
   - 用户点击"开始训练"
   - TrainingTimer.init() 初始化
   - loadMove() 加载当前动作
   - getLocalVideo() 查找本地视频

3. **播放阶段**
   - 渲染视频播放器
   - 自动静音循环播放
   - 支持手动暂停/继续
